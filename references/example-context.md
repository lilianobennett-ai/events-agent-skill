# Example: Company Context Structure

When a user wants to use this skill repeatedly, they should provide their company context as a single uploaded file or as Project knowledge. This is the structure to ask for.

The user can either paste this directly into the conversation or upload a file. Once provided, you should reference it for every event you evaluate in the session — never re-ask for context that's already been given.

## The five blocks

```markdown
# Company

**Name:** [company name]
**What we sell:** [1-2 sentences]
**Differentiation:** [why we win deals]
**Home base:** [city/country for travel cost calculations]
**Stage:** [Seed / Series A / Series B / public]

# ICP

## Tier 1 (primary buyer, 60%+ of events should target this)
- [profile 1: title + company size + industry]
- [profile 2]

## Tier 2 (influence + technical buyer)
- [profile 1]

## Tier 3 (champion / end-user)
- [profile 1]

## Named target accounts (top 20-30)
- [Company A], [Company B], [Company C]...

# Verticals

## Tier 1 (primary, 50%+ of events)
1. [Vertical A]
2. [Vertical B]

## Tier 2 (secondary, 20-30%)
3. [Vertical C]

## Verticals we DO NOT target
- [Vertical X]
- [Vertical Y]

# Competitors

## Tier 1 — Hostile direct
- [Competitor A] — [why they're direct]
- [Competitor B]

## Tier 2 — Adjacent
- [Competitor C]

## Tier 3 — Watch-list
- [Competitor D]

# Priorities

## Budget posture
[e.g., "Speaker-first preferred. Startup tiers only. Standard sponsorship requires CEO sign-off."]

## Geographic preferences
**Preferred (no flights from home base):**
- [City 1], [City 2]

**Easy travel:**
- [City 3], [City 4]

**Long-haul (only if exceptional):**
- [City 5]

## Already-committed events (do not re-recommend)
- [Event 1 — date]
- [Event 2 — date]

## Already-evaluated and rejected
- [Event X — reason]
```

## What good answers look like

When asking the user for these, ask all questions in a single message. Example:

> "Before I research events for you, I need a few details about your company. Could you give me:
>
> 1. Company name + 1-sentence description of what you sell
> 2. Top 3 buyer titles you sell to (e.g., VP Engineering, Head of Product)
> 3. Top 3 verticals you target (ranked)
> 4. Top 5 direct competitors you'd want to know if they're sponsoring an event
> 5. Home base city for travel calculations + budget posture (sponsor-heavy or speaker-first?)
>
> Once I have these I'll keep them in mind for the rest of our conversation."

If they give brief answers, that's fine — work with what they provide. You can always ask for more detail when evaluating a specific event.
